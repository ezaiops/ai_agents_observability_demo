# 05 - 抓幻觉、死循环与成本失控的告警实战

本目录是《AI Agent 可观测性（五）》文章的配套代码。

我们展示了如何将前面学到的 Trace 和 Evaluation 转化为 SRE 能够响应的 **Signal（信号）**，并编写了规则拦截器和告警 YAML 文件。

## 运行前提

1. 在 `src/` 目录下准备 `.env` 文件。建议配置兼容 OpenAI 格式的 API 从而激活真实大模型裁判；但即便不填，脚本也会平滑降级为 **MOCK 裁判模式**，以确保基于纯 Python 规则的（如死循环检测）仍可正常演示！
2. 数据集：我们在 `data/sample_traces_zh.json` 准备了 4 条极其典型的生产问题模拟数据：
   - 001：正常请求
   - 002：假成功/幻觉（配合 Judge 分数）
   - 003：无状态推进的死循环（反复查询不存在的订单）
   - 004：成本失控（一次简单查询因为未截断上下文烧掉 12.5w Tokens）

## 如何运行

本 Demo 将前四篇学到的东西串联在了一起，构成了“异步观测流水线”。

```bash
cd src
python main.py
```

## 期待看到什么？

执行结果会生动地为你演示 **“多层漏斗”** 是怎么替公司省钱和防止客诉的：
1. `trace-001`：顺利通过检查，并计算出极低的成本（Cost per Task）。
2. `trace-002`：由于异步调用裁判模型（Judge）打出了 0 分，结合底层报错被识别为【假成功/幻觉】，触发告警！
3. `trace-003`：**最精彩的部分**。纯 Python 规则 `detect_loop` 瞬间发现了这是一个“死循环”，流水线**直接将其阻断，根本没有去调用昂贵的 LLM 裁判 API 打分**，直接触发告警。
4. `trace-004`：被成本核算函数抓到了单次任务耗费 $0.6+ 的异常现象，触发告警。

除代码外，本目录下的 `rules/alerts.yaml` 还为你提供了一份可以直接套用到 Prometheus 或 Datadog 的告警规则样例。
